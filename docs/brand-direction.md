# Blog Brand Direction

## Concept
수학적 사고로 개발을 학습하는 기술 로드맵 블로그.

## Primary success criterion
방문자가 홈, 카테고리, 네비게이션을 통해 “어디서 시작하고 다음에 무엇을 읽을지” 쉽게 판단할 수 있어야 한다.

## Design principles
- 정보 전달과 가독성을 장식보다 우선한다.
- 카테고리 나열보다 학습 경로를 먼저 보여준다.
- 한국어 기술 글을 긴 호흡으로 읽기 편한 폭, 대비, 간격, 제목 계층을 사용한다.
- Minimal Mistakes/Jekyll 구조를 유지하고, 새 의존성 없이 Sass·include·layout 중심으로 개선한다.

## Learning-path groups
1. 기초 개발 역량: Python/C++ coding test, data structures, Git, environment setup, WSL2.
2. 수학·컴퓨터 과학 기반: mathematics theory, algorithm theory, cloud/computer science theory.
3. AI·비전 학습: machine learning, deep learning, computer vision, OpenCV, AI certification.
4. 실무 개발 도구와 확장: Dev Tools, Codex, Claude, Azure, web development.
5. 성능/HPC 방향성: C++, STL, CUDA/HPC roadmap.
